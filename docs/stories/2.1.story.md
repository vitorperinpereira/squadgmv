## User Story

**Como** specialist ou C-level,  
**Quero** enviar handoffs usando um contrato padrao validado,  
**Para que** agentes downstream recebam contexto completo e utilizavel.

---

## Acceptance Criteria

- [x] AC1: handoffs exigem todos os campos obrigatorios do contrato base
- [x] AC2: payloads invalidos sao rejeitados por schema
- [x] AC3: o handoff validado e persistido
- [x] AC4: a resposta de handoff tambem segue contrato estruturado

---

## Dev Notes

- `HandoffService` cria e responde handoffs.
- Os schemas vivem em `@gmv/contracts`.
- O runtime expõe `POST /api/handoffs` e `POST /api/handoffs/:id/respond`.
