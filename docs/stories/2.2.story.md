# Story 2.2: Route Handoffs between Sectors through a Managed Queue

**ID:** 2.2 | **Epic:** EPIC-2  
**Sprint:** 1 | **Points:** 5 | **Priority:** High  
**Created:** 2026-03-08

## User Story

**Como** camada de orquestracao,  
**Quero** rotear handoffs entre setores por uma fila gerenciada,  
**Para que** o trabalho avance de forma previsivel e auditavel.

---

## Acceptance Criteria

- [x] AC1: existe driver de fila para `inline` e `pg-boss`
- [x] AC2: jobs de projection e sync sao registrados e executados
- [x] AC3: os fluxos setoriais refletem handoffs com status rastreavel
- [x] AC4: retries, dead-letter e bloqueio por agentes/setores desabilitados foram fechados em ambiente real

---

## Dev Notes

- A fila atual suporta `inline` para desenvolvimento e `pg-boss` para runtime real.
- Os business flows seedam handoffs entre Marketing, Sales, Technology, Brand e Quality.
- O runtime agora bloqueia handoffs novos e seedados quando o agente alvo esta `paused` ou `disabled`, ou quando o setor nao possui capability ativa.
- O reconcile do Notion agora entra pela fila gerenciada em vez de executar sincronizacao inline.
- A operacao da fila agora expoe stats, falhas e retry manual por API e CLI.
- `workflow_runs` agora registra `queueJobId` para correlacionar execucao, retry e DLQ.
- O runtime agora faz fallback seguro para `combined` mode quando `QUEUE_DRIVER=pg-boss` mas `STATE_DRIVER=file`, evitando split-brain entre API e worker.
- O repositorio agora inclui `npm run gmv:queue:smoke`, um smoke validator real para `pg-boss` em PostgreSQL.
- O smoke validator foi executado com sucesso contra PostgreSQL do Supabase, validando DLQ, replay e reconcile real do runtime.
