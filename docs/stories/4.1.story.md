# Story 4.1: Store and Query Modular Business Memory by Domain

**ID:** 4.1 | **Epic:** EPIC-4  
**Sprint:** 1 | **Points:** 5 | **Priority:** High  
**Created:** 2026-03-08  
**Status:** Done

---

## User Story

**Como** lider de setor ou agente,  
**Quero** consultar memoria modular por dominio,  
**Para que** decisoes e workflows reutilizem contexto organizacional confiavel.

---

## Acceptance Criteria

- [x] AC1: a memoria fica separada por `brand`, `offers`, `sales`, `technology`, `operations` e `quality`
- [x] AC2: cada item de memoria mantem origem, data, contexto e relacionamento com missao ou planning item
- [x] AC3: o runtime permite consulta por dominio, missao, setor ou agente relacionado
- [x] AC4: os fluxos deixam memoria estruturada no runtime em vez de depender de memoria informal

---

## Dev Notes

- `BusinessFlowService` agora captura memoria automaticamente ao bootstrapar as verticais.
- `GET /api/memory` aceita `domain`, `missionId`, `sector`, `agent`, `tag` e `q`.
- `POST /api/memory/capture` permite registrar memoria manual adicional.
- A CLI ganhou `memory:list` e o script `npm run gmv:memory`.
