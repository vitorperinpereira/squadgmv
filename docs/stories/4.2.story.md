## User Story

**Como** Vitor Perin ou chief of staff,  
**Quero** visoes de KPI e sumarios executivos,  
**Para que** eu acompanhe saude operacional, bloqueios e riscos com rapidez.

---

## Acceptance Criteria

- [x] AC1: o runtime consolida KPIs operacionais por setor e por tipo de processo
- [x] AC2: existe geracao sob demanda de sumario executivo via API e CLI
- [x] AC3: cada KPI aponta explicitamente para processos, campanhas, missoes, entregas relevantes e objetos do Notion
- [x] AC4: o sistema torna visivel quando faltam dados para leitura confiavel

---

## Dev Notes

- `GET /api/reports/executive` consolida missoes, planning items, handoffs, approvals, validations, workflow runs e cobertura de memoria.
- O relatorio inclui `bySector`, `byProcessType`, `metricLinks`, riscos e `missingData`.
- A CLI ganhou `report:executive`, `report:generate`, `report:history` e os scripts `npm run gmv:report*`.
- A API ganhou `/api/reports/executive/generate` e `/api/reports/executive/history`.
- A recorrencia executiva agora e configuravel por `EXECUTIVE_REPORT_INTERVAL_MINUTES`, iniciada no runtime API e validada por teste automatizado.
