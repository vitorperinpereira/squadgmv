# Story 4.3: Drive Optimization Loops from Runtime Signals

**ID:** 4.3 | **Epic:** EPIC-4  
**Sprint:** 1 | **Points:** 5 | **Priority:** High  
**Created:** 2026-03-08  
**Status:** Done

---

## User Story

**Como** Vitor Perin ou lider de setor,  
**Quero** abrir e decidir iniciativas de otimizacao a partir de validacoes, memoria e sinais do runtime,  
**Para que** melhorias operacionais virem experimentos rastreaveis com aprendizado reaproveitavel.

---

## Acceptance Criteria

- [x] AC1: o runtime permite criar iniciativas de otimizacao com hipotese, origem, owner, janela de teste e criterios de sucesso
- [x] AC2: cada iniciativa pode ser decidida como `reviewing`, `adopted`, `reverted` ou `iterating`, com resultado e learnings persistidos
- [x] AC3: a missao exibe as iniciativas relacionadas e o relatorio executivo consolida o estado de otimizacao
- [x] AC4: existe operacao por API e CLI para listar e abrir iniciativas sem depender de UI

---

## Dev Notes

- `OptimizationService` centraliza criacao, decisao e listagem de iniciativas.
- O runtime ganhou `GET /api/optimizations`, `POST /api/optimizations` e `POST /api/optimizations/:id/decision`.
- A CLI ganhou `optimizations:list` e `optimization:create`, com atalho `npm run gmv:optimizations`.
- O detalhe da missao agora retorna `optimizationInitiatives` e o relatorio executivo passou a consolidar o bloco `optimization`.
